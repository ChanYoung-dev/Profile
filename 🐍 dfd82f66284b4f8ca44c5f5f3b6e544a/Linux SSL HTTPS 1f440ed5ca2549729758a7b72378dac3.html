Linux SSL HTTPS

Web encryption 에 설명한 암호화 방식들의 단점들을 해결하기 위해 SSL 방식이 나왔다

SSL

대칭키 방식 : 암호화하고 푸는 것이 쉽지만 키 배송문제 발생

공개키 방식 : 키 배송 문제는 없지만 암호화하고 푸는 것이 어렵다는 단점

→ 공개키 암호화는 대칭키를 보내는 용도에만 사용

이러한 방법을

표준보안방식 SSL 이라고한다

SSL 의 방법

https(SSL) port :443

SSL : Secure Socket Layer

  • 1973년 netscape사에 의해 개발된 통신 규약
  • 세션계층에서 적용되며 telnet, ftp, http 등의 프로토콜의 안정성 보장
  • 서버인증, 클라이언트 인증, 기밀성 보장

하지만 내가 접속한 사이트가 진짜인지 알수가 없다

이를 해결하기위해 제 3의 인증 기관(CA)가 생겼다

HTTPS 실습

1 . 개인키 만들기 : 키 길이 2048, rsa방식으로 암호

openssl genrsa -out private.key 2048

  1. CSR(Certificagte Signing Request) 생성

CSR을 CA에 제출하고 서명된 인증서를 받아야한다

하지만 실습용이므로 자필 서명인증서(Self-Signed Certificate)를 생성하여 사용한다

자필 서명인증서 생성

openssl req -new -key private.key -out cert.csr

cert.csr을 만든다

  1. 인증서만들기

openssl x509 -req -signkey private.key -in cert.csr -out cert.crt

인증서의 개인키(private.key)로 cert.csr요굿서 내용을 cert.crt의 인증서로 만든다

  1. 안전한곳으로 옮기기

mv /root/cert.crt /etc/pki/tls/certs

mv /root/private.key /etc/pki/tls/private/

  1. 개인키 자기자신만 사용

chmod 600 /etc/pki/tls/private/private.key

  1. apache에서 ssl을 사용하기위한 모듈 설치

yum -y install mod_ssl

  1. apache에서 ssl관련설정

vi /etc/httpd/conf.d/ssl.conf
