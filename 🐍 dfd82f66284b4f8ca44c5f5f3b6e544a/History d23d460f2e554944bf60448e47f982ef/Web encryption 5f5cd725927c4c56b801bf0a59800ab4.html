암호화 이론

일방향암호화 : md5, sha256, sha512, 해쉬알고리즘

  • 암호화과정만 있고 복호화과정이 없다
  • A를 암호화를 화면 항상 일정한 암호화값이 나온다

    일방향 암호화의 대표인 md5암호화로 예를 들면 항상 1234를 암호화하면

    81DC9BDB52D04DC20036DBD8313ED055

    으로 일정하다

  • 엄밀히 말하면 복호화과정이 상당히 복잡하다
  • 그렇기때문에 비밀번호찾기를 누르면 비밀번호를 알려주는 것이 아니라 비밀번호 재설정을 하는 것이다

양방향암호화 :

대칭키(비밀키) 암호화

  • 대칭키 1개 사용
  • 암호화, 복호화에 같은 키를 사용
  • 보내는 사람과 받는 사람 둘다 같은 키로 암호화하고 복호화하므로 대칭키라 표현
  • 그러므로 수신자와 송신자는 키를 교환해야한다
  • 공유한 키(비밀키)는 비밀로 유지
  • 비대칭키 암호화 통신보다 1000배이상 빠름
  • 알고리즘 : DES, 3DES, AES 등등

단점

키배송의 취약점(중간에 비밀키를 해킹당하면 답이 없다)

비대칭키(개인키, 공개키) 암호화 -> 키 2개

원리

개인키로 암호화한 것은 오직 공개키로 해독이 가능하다.

공개키로 암호화한 것은 오직 개인키로 해독이 가능하다.

  • 공개키는 누구나 가질 수 있다
  • 개인키는 오직 자기자신만 갖고 비밀로 유지한다
  • 공개키만 공개하면 된다
  • 암호/복호화에 동일한 알고리즘을 사용하지만 서로 다른 키를 이용한다

  • 인증 기능을 가지고 있다. (전자 서명)
  • 알고리즘 : RSA
    • 수학적인 방법으로 서로 다른 한쌍의 암호화키(A-B)를 만든다
    • 키 A로 암호화하면 B로만 풀 수 있다
    • 키 B로 암호화하면 A로만 풀 수 있다
    • 각 키는 매우 큰 숫자이므로 한 개의 키로 다른 키를 알아내기 어렵다
    • 키 A만 잘 숨겨 놓으면 된다. 이러한 키가 개인키다
    • 키 B는 아무한테나 알려줘도 된다. 이러한 키가 공개키이다

    순서

단점

  • 공개키 암호화는 너무나 복잡해서 암호화하고 풀기엔 어렵다
  • 그렇기때문에 시간이 걸리고 CPU도 좋아야 하는 단점이 있다

위의 각 양방향 암호화에 대한 단점들을 해결하기 위해

두가지 형태를 혼합한 형태인 SSL 방식이 나타났다

SSL에 관하여 : Linux SSL HTTPS
