IP 프로토콜의 한계

  • 비연결성

패킷 받을 대상이 없거나 서비스 불능 산태여도 패킷전송을 한다

  • 비신뢰성

패킷이 중간에 어떤 서버가 종료되거나 오류가나 패킷이 사라짐

순서대로 패킷이 안올때 ->1500바이트씩 메세지를 나눠 보내는데 나누어진 메시지가 같은 순서대로 안오면 문제가생김

  • 프로그램

게임, 음악이런걸 동시에 할때 IP는 하나인데 구분은 어떻게 할것인가

결론

복잡한 인터넷망에서 메세지를 보내기위해선 인터넷프로토콜(ip)가 있어야한다

IP의 문제

하지만 ip프로토콜만으론 메세지가 잘도착했는지 신뢰하기도 어렵고, 포트라는 개념이 없다. 메시지 순서도 꼬일수도있다.

TCP/UDP

이러한 문제를 tcp가 해결해준다. udp는 거의 ip와 같고 포트정도 추가된 정도다. 필요하면 udp프로토콜위에서 애플리케이션에 맞게 확장시킬수있다. TCP는 연결설정을 하는 3-handshaking때문에 비연결성인 UDP에 비해 시간이 오래걸리기때문에 요즘은 UDP를 많이 사용한다.

Port

port는 같은 IP안 에서 동작하는 애플리케이션을 구분하기위해서 사용한다 쉽게말해서 ip가 아파트라고 하면 port가 동호수다

DNS

ip는 늘변할수있고 외우기어렵지만 dns에 도메인 이름으로 등록해주면 ip가 바뀌어도 도메인 이름만 적용함으로써 사용할수있다.
