Proxy

  • 네트워크 중간에서 대리인 역할을 하는 서버
  • 네트워크 연결 중간에서 거쳐가는 서버로 중간에서 각각의 시스템에 데이터를 넘겨받아 전달한다.
  • 한번 전송한 데이터는 자기 시스템에 캐쉬로 저장하여 똑같은 데이터 전송이 필요할시 캐쉬를 이용하여 속도를 향상시킨다

프록시 툴 설치

yum -y install squid

squid.conf 설정 수정

vi /etc/squid/squid.conf

http_access allow all #http_access allow localhost 
# 로컬에서만 접속가능이었지만 바꿔준다 외부 ip를 허용하도록
cache_dir ufs /var/spool/squid 1000 16 256 # 캐쉬용량설정

Proxy 포트를 열어준다

firewall-cmd --permanent --add-port=3128/tcp

firewall-cmd —reload

설정 적용

[root@localhost ~]# systemctl restart squid
[root@localhost ~]# systemctl enable squid

<DHCP 동작방식>

DHCP Discovery : PC가 DHCP를 찾기 위해 브로드캐스팅으로 날린다

DHCP Offer : 여러개의 DHCP서버가 각각 유니캐스팅으로 DHCP서비스를 제공해줄수있다고 PC한테 보낸다

DHCP Request : 여러개의 DHCP중에 선택하여 특정 DHCP를 선택한다고 브로드캐스팅한다.

DHCP Acknowledge : 특정 DHCP가 DHCP를 할당한다

dhcp 설정파일 변경

vi /etc/dhcp/dhcpd.conf

아래내용을 추가

subnet  10.0.2.0        netmask 255.255.255.0 { #dhcp가 소속하고있는 현재네트워크
				# 실제로 클라이언트에게 알려줄 주소
        option  routers 10.0.2.1;
        option  subnet-mask     255.255.255.0;
        range   dynamic-bootp   10.0.2.10       10.0.2.100; # 임대해줄 ip주소 범위
        option  domain-name-servers     8.8.8.8;
        default-lease-time      10000; # 최소임대시간(초단위)
        max-lease-time  50000; # 최대임대시간(초단위)
}

설정적용

[root@localhost ~]# systemctl restart dhcpd
[root@localhost ~]# systemctl enable dhcpd

방화벽열기

[root@localhost dhcpd]# firewall-cmd --add-service dhcp
success
[root@localhost dhcpd]# firewall-cmd --reload
