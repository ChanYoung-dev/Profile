서비스 유닛 정보
Loaded - 서비스 유닛이 메모리에 로드되었는지의 여부
Active - 서비스 유닛이 실행 중인지의 여부 확인
Main PID - 기본 프로세스 ID
status - 서비스에 대한 추가 정보

systemctl 출력의 서비스 상태
active(running) - 프로세스 정상 실행중
active(exited) - 초기 설정을 완료하고 대기중
active(waiting) - 인터럽트 신호 대기
inactive - 실행 중이 아님
enabled - 부팅할 때 시작
disabled - 부팅할 때 시작되지 않도록 설정

- 서비스 재시작 중 발생할 수 있는 문제를 방지

로컬 사용자 및 그룹 관리
사용자 및 그룹
- 사용자와 그룹은 시스템에서 필수적인 요소
- 사용자가 없으면 시스템에 로그인할 수도 없고 사용할 수도 없음

사용자
- 로그인 대상
- 접근 제어 기준으로 분류(1차적)
- 다른 사용자와 명령을 실행할 수 있는 프로그램 간의 보안 경계 제공
 - 일반 사용자로 root 디렉토리 내 파일 리스트나 /etc/shadow 파일
내용을 볼 수 없음
- /etc/passwd 파일에 사용자에 대한 정보 저장
- 슈퍼유저, 시스템 사용자, 일반 사용자로 분류
 - 0번 : root
 - 1 ~ 999번 : 시스템 사용자, 프로세스에서 사용
  - 각각의 프로세스에 맞춰 시스템에 대한 엑세스 제한
 - 1000번 이상 : 일반 사용자, 작업에 사용

그룹
- 사용자 하나하나에 권한 부여를 하기엔 작업이 너무 많아짐
 - 그룹으로 제어
- 사용자들의 집합
- /etc/group 파일에 그룹에 대한 정보 저장
- 접근 제어 기준으로 분류 (2차적)

/etc/passwd 파일 확인
# cat /etc/passwd | grep root
root:x:0:0:root:/root:/bin/bash
---- - - - ----- ----- ----------
  1  2 3 4   5    6         7

1. root - 사용자의 이름, 로그인 할 때 사용
2. x - 패스워드 (현재 /etc/shadow 파일에 저장)
3. 0 : 사용자의 UID, 시스템에서 사용자를 식별하는 번호
4. 0 : 그룹의 GID, 시스템에서 그룹을 식별하는 번호
5. root : 메모(정보) or 코멘트
6. /root : 사용자의 홈디렉토리
7. /bin/bash : 사용 중인 쉘

/etc/group 파일 확인
# cat /etc/group | grep root
root:x:0:
---- - - --
  1  2 3 4

1. root : 그룹 이름
2. x : 그룹 패스워드 (현재 /etc/gshadow 파일에 저장)
3. 0 : 그룹 아이디, GID
4. (공백) : 이 그룹을 보조 그룹으로 지정하고 있는 사용자의 목록

사용자의 정보 확인
# id

슈퍼유저(root)
- 다양한 권한을 가진 사용자
- uid가 0번으로 고정
- 일반 사용자에서 root 사용자로 로그인할 때 "su -" 사용

sudo
- 관리자 권한으로 명령 실행
- sudo 사용할 경우 /var/log/secure 파일에 로그 기록

관리자 권한을 갖는 사용자 vs 일반 사용자
      sudo 사용 가능	 sudo 사용 불가

사용자 및 그룹 관리
- 관리자 권한 필요
- 사용자
useradd : 사용자 생성
# useradd user1
# tail -n 1 /etc/passwd
결과) user1:x:1000:1000::/home/user1:/bin/bash
# tail -n 1 /etc/group
결과) user1:x:1000:
- 일반 사용자 생성 시 기본 그룹도 생성

많이 사용하는 u, g, G, s, D 옵션
u 옵션
- 사용자의 아이디를 지정
# useradd -u 1004 user2
# tail -n 1 /etc/passwd
결과) user2:x:1004:1004::/home/user2:/bin/bash
- UID를 지정하여 생성시에 똑같은 GID도 생성

g 옵션
- 기본 그룹 지정
# useradd -g 1004 user3
# tail -n 1 /etc/passwd
결과) user3:x:1005:1004::/home/user3:/bin/bash
- 사용자를 생성할 때 그룹을 지정할 시 있는 그룹으로 지정해야 함
- 사용자 생성시 가장 마지막 번호 다음 숫자로 UID를 부여
- user3 에 해당하는 기본 그룹은 생성 X

G 옵션
- 보조 그룹 지정
# useradd -G 1000 user4
# tail -n 1 /etc/passwd
결과) user4:x:1006:1006::/home/user4:/bin/bash

보조그룹 지정 확인
# tail /etc/group
결과)
...
user1:x:1000:user4
...

s 옵션
- 쉘 지정
# useradd -s /bin/sh user5
# tail -n 1 /etc/passwd
user5:x:1007:1007::/home/user5:/bin/sh
                                         --------

D 옵션
- 사용자 생성시 기본 정보 확인
# useradd -D

g 옵션

G 옵션
- 보조 그룹 변경
# usermod -G 1007 user1

s 옵션

선행 작업

# passwd user1
- 먼저 user1의 비밀번호를 변경(root 비밀번호와 다르게 설정)

# vim /etc/sudoers

- wheel 그룹에 넣기

- 일반 사용자 비밀번호 변경
# passwd user2
- 먼저 user2의 비밀번호를 변경(root 비밀번호와 다르게 설정)

사용자 암호 관리
- /etc/shadow 디렉토리에 정보가 저장
# head -n 1 /etc/shadow
root:$6$Kuu3cydETuBmFPoN$0UXdEHZliO3hxkkrjp72gpR6WVdtfdkBmeEgF0
7NHGoHePMEYX4bc22/vZsKf.Sju0utCM8/CLerebM2F7iQx/::0:99999:7:::

1. root - 사용자명
2. 암호
- $6 : 해시 알고리즘 종류, sha512
- $Kuu3cydETuBmFPoN : 추가된 랜덤값(solt값)
- $0UXdEHZliO3hxkkrjp72gpR6WVdtfdkBmeEgF0
7NHGoHePMEYX4bc22/vZsKf.Sju0utCM8/CLerebM2F7iQx/ : 결과값
3. (공백) - 암호가 변경된 마지막 날, 1970년 1월 1일 이후로 설정되며 UTC 시간대로 계산
4. 0 - 사용자가 암호를 다시 변경하기 위해 경과해야 하는 최소 일 수
5. 99999 - 암호 만기일, 1970년 1월 1일 기준으로 설정
6. 7 - 암호 만기일 전 경고 기간
7. (공백) - 암호가 만료되어도 로그인 할 수 있는 기간
8. (공백) - 계정 만료일, 1970년 1월 1일 기준으로 설정
9. (공백) - 사용 X (예약)

/etc/shadow 파일에서 2번째 필드
* - 패스워드 잠금 상태, 별도의 인증 방식 필요
!! - 암호 설정 x
- root 사용자에서 암호 설정이 안 된 일반 사용자로 로그인 가능
- 일반 사용자에서 암호 설정이 안 된 일반 사용자로 로그인 불가

chage : 비밀번호와 관련된 설정을 할 수 있음
옵션
l 옵션 : 설정 확인(소문자 l)
# chage -l user1

m 옵션 : 최소 사용 기간 설정
# chage -m 3 user1
# chage -l user1   <--- 확인

M 옵션 : 최대 사용 기간 설정
# chage -M 100 user1
# chage -l user1   <--- 확인

W 옵션 : 암호 만료 전 경규 기간 설정
# chage -W 10 user1
# chage -l user1   <--- 확인

I 옵션 : 비활성 기간(암호가 만료되어도 로그인 할 수 있는 기간) (대문자 i)
# chage -I 30 user1

파일에 대한 엑세스 제어
접근제어
- 권한과 관련있는 부분

권한 변경
fileA를 소유자는 rwx 권한, 그룹은 rw-, 기타사용자 아무 권한 없게 설정하기
# chmod u+x fileA
# chmod g+w fileA
# chmod o-r fileA
# ls -l fileA
결과) -rwxrw----. 1 root root 0 Jan 11 08:27 fileA

fileC를 소유자, 그룹, 기타 사용자의 권한에서 읽기 권한을 빼기
# chmod a-r fileC

fileB를 소유자는 rwx 권한, 그룹은 rw-, 기타사용자 아무 권한 없게 설정하기
# chmod 760 fileB

특수권한
- 세 가지 권한 이외에도 추가 액세스 관련 기능 제공
- 소유자에서의 s, 소유그룹에서의 s, 기타사용자의 t

소유자에서의 s
- setuid
- 파일을 실행할 때 파일의 소유자 권한으로 실행
# ls -l /bin/passwd
-rwsr-xr-x. 1 root root 27856 Mar 31  2020 /bin/passwd
    -

소유 그룹에서의 s
- setgid
- 디렉토리에서 설정할 수 있는 기능
- 디렉토리에 새로 생성된 파일에는 디렉토리의 소유 그룹과 일치하는 소유 그룹이
설정됨
# ls -ld /run/log/journal/
drwxr-sr-x. 3 root systemd-journal 60 Jan 12 19:38 /run/log/journal/
        -

기타 사용자에서의 t
- sticky bit
- 디렉토리에서만 사용
 - 특정 디렉토리를 누구나 자유롭게 사용할 수 있게 하기 위함(공용 디렉토리)
- 쓰기는 아무나 가능하나 생성딘 파일은 타 사용자가 강제로 제거 불가
# ls -ld /tmp
drwxrwxrwt. 12 root root 4096 Jan 12 22:29 /tmp

setuid 설정
# chmod u+s fileA
# ls -l fileA
-rwsrwx--x. 1 root root 0 Jan 11 08:27 fileA
    -

setgid 설정
# chmod g+s dirA
# ls -ld dirA
drwxr-sr-x. 2 root root 6 Jan 13 01:25 dirA
        -

stickybit 설정
# chmod o+t dirA
# ls -ld dirA
drwxr-sr-t. 2 root root 6 Jan 13 01:25 dirA
            -

실행 권한이 없는 상태에서 setuid, setgid, stickybit 설정을 하게 되면
각각 대문자 S, 대문자 S, 대문자 T 가 표기됨
