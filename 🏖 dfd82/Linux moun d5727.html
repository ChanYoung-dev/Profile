rpm

  • 패키지를 관리하는 용도로 사용
  • 로컬에 파일이 있어야 설치 가능
  • 파일 형식 : 이름-버전-릴리즈 버전-아키텍처.rpm
  • Windows에서 .exe 파일과 흡사
  • 컴파일되어 설치할 실행파일, 설정파일, 라이브러리 등을 하나로 묶어놓은 파일
  • 설치된 패키지를 모두 나열 rpm -qa - q 옵션 : 쿼리, 특정 패키지에 대한 정보를 확인할 때 사용해야 하는 필수 옵션 - qa 옵션 : 설치된 패키지를 모두 나열 - noarch : 패키지가 python, perl 같은 스크립트같은 경우 32비트, 64비트 상관 없이 동작하므로 X86_64나 i686 대신 noarch 라고 표시됨
  • rpm 파일 설치 rpm -ivh httpd-2.4.6-97.el7.centos.2.x86_64.rpm 파일의 위치를 알아야 함

    • 패키지 간 의존성 문제 해결 (ex. A 패키지를 설치하기 전에 필요한 B 패키지까지 모두 한 번에 설치 가능)

파일 시스템

파일 시스템은 해당 저장 공간에서 작업할 때 적용되는 옵션들을 설정해두는 것(압축, 공유, 작업 설정 등)

  • 계층 구조도 파일 시스템과 관련이 있음

  • /dev 스토리지가 포함된 장치와 관련된 디렉토리
    • 스토리지 정보 확인 lsblk
    • 디스크 파티셔닝을 위한 선행 작업(디스크 추가) 1. 가상머신 시스템 종료 2. 가상머신 선택 후 [설정] 클릭 3. [저장소] 항목 클릭 4. [컨트롤러 : SATA] 항목 맨 왼쪽 아이콘 클릭 - 네모디스크에 "+" 모양이 합쳐진 아이콘 5. [만들기] 클릭 6. [VDI] 선택 후 [다음] 클릭 7. [동적 할당] 선택 후 [다음] 클릭 8. 용량 3GB 선택 후 [만들기] 클릭 9. 만든 하드디스크를 [선택]하여 적용
  • 파일시스템 검사 : df
    • 로컬 및 원격 파일 시스템 장치 및 가용 공간에 대한 개요 확인
    • h 옵션을 사용하여 가독성 up
  • 특정 디렉토리의 디스크 사용 용량 확인 : du
    • h 옵션을 사용하여 가독성 up

sftp

  • ssh처럼 접속(대화형)
  • 현재 시스템이 포커스, 원격 시스템에서 작업
  • 파일의 위치를 파악해서 해당 파일을 가져올 수 있음
  • sftp 접속 sftp root@192.168.56.10 포커스 : 192.168.56.10 가상머신의 /root
  • 192.168.56.10의 /root/anaconda-ks.cfg를 192.168.56.11의 /root 디렉토리로 복사 sftp> get anaconda-ks.cfg
    • 복사된 파일 확인 sftp> exit
    • 파일이 추가됐는지 확인 : ls 결과) anaconda-ks.cfg dirA fileA
  • 192.168.56.10의 /root/fileA를 192.168.56.11의 /root 디렉토리로 복사
    • 원격 접속

      sftp root@192.168.56.10

    • put 사용

cron : 주기적인 작업 예약

예약된 작업을 반복 실행 crond 데몬이 작업을 제어(crontab 구성 파일 해석)

  • crontab 파일 확인 vi /etc/crontab crontab 파일에 설명을 확인 
[root@servera ~]# vi /etc/crontab

SHELL=/bin/bash
PATH=/sbin:/bin:/usr/sbin:/usr/bin
MAILTO=root

# For details see man 4 crontabs

# Example of job definition:
# .---------------- minute (0 - 59)
# |  .------------- hour (0 - 23)
# |  |  .---------- day of month (1 - 31)
# |  |  |  .------- month (1 - 12) OR jan,feb,mar,apr ...
# |  |  |  |  .---- day of week (0 - 6) (Sunday=0 or 7) OR sun,mon,tue,wed,thu,fri,sat
# |  |  |  |  |
# *  *  *  *  * user-name  command to be executed
* * * * * root echo "test" > test.txt # <- 작성

* * * * * root echo "test" > test.txt

  • root 사용자가 echo "test" > test.txt 실행
  • 매분 매시 매일 매월 매요일마다 실행 -> 1분마다 계속 실행
  • 현재 시간을 기준으로 실행 - 분 자리에 * 가 표기될 경우 현재 시간 기준

* * * * * user1 echo "test" > test.txt

  • /home/user1 디렉토리에서 echo "test" > test.txt 실행
  • 매분 매시 매일 매월 매요일마다 실행 -> 1분마다 계속 실행

0 1 * * * root echo "test" > test.txt

  • 매일 오전 1시에 실행

0 1 * * 0 root echo "test" > test.txt

  • 매월 매일 일요일 오전 1시에 실행

0 1 1 * 0 root echo "test" > test.txt

  • 매월 1일 일요일 1시에 실행
  • 1월 1일이 월요일? 실행 X

0 1 1 2 0 root echo "test" > test.txt

  • 2월 1일 일요일 1시에 실행
  • 2월 1일이 월요일? 실행 X

0/3 1 * * * root echo "test" > test.txt

  • 매일 오전 1시에 3분마다 실행
  • 2시가 되면 실행 X
