umask
- 현재 설정된 권한 값을 확인하는 명령어
# umask
결과) 0022

디렉토리 기본값 : 777
파일 기본값 : 666

기본 권한 = 기본값 - umask

umask 기본값은 /etc/profile 에 저장

umask 값을 변경하고 싶은 경우 /etc/profile 파일 내
하단에 umask [숫자] 를 입력한 후 저장 및 재부팅 진행

소유권 바꾸기
chown
# chown user1 fileA
- fileA의 소유자가 user1으로 변경

# chown :user1 fileA
- fileA의 소유그룹이 user1으로 변경

# chown root:root fileA
- fileA의 소유자와 소유그룹이 root로 변경됨

ssh
- 원격 접속 프로토콜
- ssh 프로토콜을 사용하여 안전하지 않은 네트워크를 암호화된 보안
방식으로 통신
- telnet 등 다른 프토토콜들도 있지만 안정성을 위해 ssh 사용

사용 형식)
ssh [호스트명]
ssh [사용자명]@[호스트명]
ssh [사용자명]@[ip 주소]
ssh [사용자명]@[ip 주소] [명령어]
- 일회성 명령어를 입력한 후 로그아웃

# ssh root@192.168.56.11
- 가상머신 2번에 ssh 접속

/etc/ssh/sshd_config
- ssh 데몬 설정파일

PermitRootLogin
- yes 설정 시 루트로 바로 로그인 가능
- no 설정 시 루트로 바로 로그인 불가

PasswordAuthentication
- yes 설정 시 모든 사용자에 대해서 패스워드 인증 방식 허용
- no 설정 시 모든 사용자에 대해서 키 기반 인증 방식 허용 (패스워드 X)

로그
- 어디에서 문제가 발생했는지, 무슨 작업을 했는지를 확인하기 위한
목적으로 기록
- 특정 서비스에 의해서 기록
- 이벤트 발생시
이벤트 -> systemd-journald -> rsyslogd
	    1차 가공	   2차 가공
	 (/run/log/journal)	   (/var/log)
- run 디렉토리는 휘발성 디렉토리

비교
	          journal		syslog
저장 형태	      바이너리 형식	        ASCII 텍스트 형식
저장 유지	          휘발성          비휘발성(일정 시간 지난 후 삭제)
확인 명령어      journalctl	    일반적인 확인 명령어
			    (cat, tail, head, vim, ...)

journal 파일 내용 확인
# head -n 3 /run/log/journal/1cf12d98f4fc8f469f4669f405287f50/system.journal
- 바이너리 형식이기 때문에 사용자가 육안으로 식별하기 힘듬

journal 로그 확인
# journalctl
- 2차 가공된 로그보다 많은 로그 확인 가능

syslog 파일 확인
# ls /var/log

secure 로그 파일 확인
# tail /var/log/secure

내용 중 마지막 열 줄을 출력한 후 실시간 모니터링
# tail -f /var/log/secure
- ctrl + c로 빠져나오기

syslog 우선 순위
emerg - 시스템이 불능상태에 빠졌을 경우
alert - 시스템이 멈출 수 있는 치명적인 오류인 경우
crit - 시스템에 영향이 있을 수 있는 경우
err - 명령어가 잘못된 경우
warning - 설정은 되지만 조건에 충족하지 못하는 경우
notice - 에러가 아닌 알람에 대한 메세지
info - 단순한 프로그램에 대한 메세지
debug - 디버깅에 대한 메세지

networkmanager
- 리눅스 네트워크 관리자
1. nm-connection-editor
- 그래픽 환경에서만 사용가능한 설정 변경 툴

2. nmtui
- ipv6 지원 X

3. nmcli
- cli 환경에서 사용하는 명령어
- 네트워크 관리는 수퍼유저 권한 필요
- 명령어를 입력하여 설정을 변경하는 것이므로 네트워크 서비스를 재시작해주는 
것이 아닌 nmcli con reload 명령어를 입력해주어야 함

네트워크 장치로 연결 정보 확인하기
# nmcli dev status

네트워크 장치 세부 정보 확인
# nmcli dev show

연결 설정 정보 확인
# nmcli con show

특정 연결 설정 정보 확인
# nmcli con show 'ens1'

dhcp 자동으로 연결
# nmcli con add con-name ens3 type ethernet ifname enp0s8
# nmcli con up ens3

네트워크 수동 설정 추가
# nmcli con add con-name ens4 type ethernet ifname enp0s8 \ 
ipv4.method manual ipv4.addresses 192.168.56.10/24 \
ipv4.gateway 192.168.56.1 ipv4.dns 8.8.8.8
# nmcli con up ens4

네트워크 연결 설정 변경
# nmcli con mod ens4 ipv4.addresses 192.168.56.12
- 해당하는 부분을 바꿔주면 됨

설정 변경 후 해줘야 할 것
# nmcli con reload
# nmcli con down ens4
# nmcli con up ens4
- 설정 변경 후 설정을 다시 로드해주고 해당 연결을 해제 후 다시 연결
- 다른 ip로 연결해줄 시 putty 연결이 끊어짐

192.168.56.12 putty 연결 후
# nmcli con up ens2

192.168.56.10 putty 연결

네트워크 연결 제거
# nmcli con del ens3
# nmcli con del ens4
